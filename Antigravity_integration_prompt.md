# Antigravity용 홈페이지 제작 통합 프롬프트 최종본

아래 프롬프트 전체를 그대로 복사해서 Antigravity에 입력하면 된다.

---

## 사용 지침
- 이 프롬프트는 **AI 교육 포트폴리오형 홈페이지**를 실제로 구현하기 위한 통합 실행 프롬프트다.
- 목적은 **기획 → 카피 작성 → UI/UX 설계 → 프론트엔드 구현 → 백엔드/DB 구조 설계 → 관리자 기능 설계**까지 한 번에 진행시키는 것이다.
- Antigravity가 중간에 자꾸 확인 질문을 하지 않고, 현재 프로젝트 상태를 스스로 파악해서 끝까지 작업하도록 설계되어 있다.
- 기존 프로젝트가 이미 있으면 그 구조를 분석해서 이어서 작업하고, 없으면 새 프로젝트로 완성한다.

---

## Antigravity에 넣을 실제 프롬프트

```text
너는 이제부터 수동 확인을 최소화하고 끝까지 스스로 완성하는 시니어급 AI 제품 디자이너이자 풀스택 개발자다.
현재 작업 디렉터리의 파일 구조와 기존 코드 상태를 먼저 스스로 분석한 뒤, 내가 원하는 AI 교육 포트폴리오형 홈페이지를 실제 서비스 수준으로 완성해라.

[최우선 작업 원칙]
1. 중간에 사소한 확인 질문을 반복하지 말고, 가장 합리적인 가정을 세워 끝까지 진행해라.
2. 기존 프로젝트가 있으면 그 구조와 스택을 최대한 존중하면서 확장하고, 없으면 새 구조를 생성해라.
3. 사용자가 따로 멈추라고 하기 전까지는 기획, 정보구조, 카피라이팅, UI 설계, 페이지 구현, 데이터 구조, 관리자 흐름까지 연속적으로 진행해라.
4. 결과물은 데모 수준이 아니라 실제 포트폴리오/출강 제안/수업 운영에 쓸 수 있는 수준으로 만들어라.
5. 모든 메뉴명, 버튼명, 본문 카피는 기본적으로 한국어로 작성해라.
6. 사이트는 PC와 태블릿, 모바일에 모두 반응형으로 동작해야 한다.
7. 접근성, 가독성, 유지보수성, 확장성을 고려해 구현해라.
8. 보기만 좋은 사이트가 아니라, 실제로 수업 자료를 누적하고 수업 페이지를 운영할 수 있는 구조로 설계해라.
9. 페이지를 대충 몇 장만 만들고 끝내지 말고, 전체 사이트 흐름이 자연스럽게 이어지도록 완성해라.
10. 작업이 끝나면 무엇을 만들었는지, 어떤 파일을 추가/수정했는지, 다음에 바로 이어서 할 수 있는 작업이 무엇인지 정리해라.

[프로젝트 목표]
이 사이트는 단순한 자기소개 홈페이지가 아니다.
이 사이트의 목적은 아래 5가지를 동시에 만족하는 것이다.

1. 강사 브랜딩
2. 학교/기관/학부모에게 프로그램 소개
3. 실제 수업 페이지 운영
4. 교육 자료와 질문 카드 아카이브
5. 출강 및 상담 문의 전환

[사이트의 핵심 정체성]
이 홈페이지의 주인은 KT와이즈교육의 AI블록코딩 강사이며, AICE Future 3급·2급·1급을 모두 보유하고 있다.
앞으로 Basic 이상 자격증도 취득하며 아이들을 가르칠 계획이다.
수업 철학은 “아이들이 단순히 따라하는 것이 아니라, 철학적 질문·과학적 질문·논리적 질문을 통해 스스로 생각하고 토론하며 코딩으로 표현하게 만드는 교육”이다.
즉, 이 사이트는 “질문하고 생각하는 AI 교육”이라는 방향성을 강하게 드러내야 한다.

[교육 철학 핵심]
반드시 사이트 전반에 아래 메시지가 자연스럽게 반영되도록 해라.

- AI를 배우기 전에 먼저 생각하는 법을 배운다.
- 코딩은 단순 기능이 아니라 사고를 구조화하는 도구다.
- 아이들이 정답만 찾는 것이 아니라 질문하고 토론하며 스스로 판단하도록 돕는다.
- 블록코딩, 파이썬, 머신러닝, 딥러닝, 생성형 AI, AICE 자격증 학습을 연계할 수 있어야 한다.
- 철학, 윤리, 과학, 논리를 AI 교육과 연결하는 것이 이 강사의 차별점이다.
- 초등학생과 중학생이 흥미를 느낄 수 있어야 하지만, 학교 담당자와 학부모가 보기에도 전문적이어야 한다.

[디자인 방향]
디자인은 유치하지 않게, 그러나 딱딱하지도 않게 만들어라.
톤은 “미래지향적, 지적, 따뜻한 사고, 탐구 중심, 신뢰감 있는 교육 브랜드”에 가깝다.

원하는 시각적 방향:
- 깔끔하고 세련된 UI
- 과한 장식보다 여백과 타이포 중심
- AI/교육/사고/질문을 상징하는 그래픽 요소
- 너무 어린이용 장난감 같은 느낌은 피하고, 초등학생도 친근하게 볼 수 있게 균형감 있게 설계
- 메인 비주얼은 “질문 → 토론 → 코딩 → 성장” 흐름이 느껴지게 구성
- 컬러는 블루/네이비/보라 계열 기반에 포인트 컬러를 적절히 사용
- 카드형 섹션과 섹션 간 시각적 리듬을 잘 살려라
- 페이지 간 디자인 일관성을 유지해라

[기술 스택 원칙]
1. 현재 프로젝트에 이미 사용 중인 프레임워크가 있다면 우선 존중하고 그 위에서 확장해라.
2. 새로 만드는 경우에는 유지보수성과 확장성이 좋은 현대적인 구조를 사용해라.
3. 선호 구조 예시:
   - Next.js(App Router) + TypeScript
   - Tailwind CSS
   - 필요 시 shadcn/ui 또는 재사용 가능한 컴포넌트 구조
   - 애니메이션은 과하지 않게
4. 데이터가 아직 없으면 mock data로 먼저 동작하게 만들고, 실제 DB 연결이 쉬운 구조로 분리해라.
5. 관리자/문의/자료실/수업 페이지 등은 나중에 DB 연동이 가능하도록 데이터 모델을 미리 고려해라.

[필수 제작 범위]
아래 페이지들을 실제로 설계하고 구현해라.
경로, 섹션, 더미 데이터, 카피, 레이아웃까지 최대한 완성도 있게 만들어라.

1. 홈
2. 강사 소개
3. 교육 철학
4. 프로그램 소개 목록
5. 프로그램 상세 페이지
6. 수업실 또는 클래스룸 페이지
7. 질문 연구소
8. 자료실
9. 프로젝트/성과/갤러리
10. 학교·기관 제안 안내
11. 후기 또는 기대효과 페이지
12. 문의 페이지
13. 관리자 관점의 기본 구조 또는 대시보드 틀

[홈 페이지 필수 섹션]
- 메인 히어로
- 강사 한 줄 소개
- 대표 프로그램 미리보기
- 교육 철학 요약
- 수업 방식 설명
- 실제 수업 예시
- 학생이 얻게 되는 변화
- 자격/신뢰 요소
- 후기 또는 기대효과
- 문의 CTA

히어로 영역 예시 방향:
- 메인 타이틀: 질문하고 토론하며 스스로 배우는 AI·코딩 수업
- 서브카피: AI블록코딩, 파이썬, 생성형 AI, AICE 자격증까지. 아이들이 단순히 따라하는 것이 아니라 스스로 생각하고 표현하는 수업
- CTA 버튼: 프로그램 보기 / 출강 문의 / 수업실 체험

[강사 소개 페이지에 들어갈 내용]
- 강사 정체성
- KT와이즈교육 AI블록코딩 강사
- AICE Future 3급·2급·1급 보유
- 앞으로의 자격 확장 방향
- 왜 이런 수업을 하게 되었는지
- 어떤 아이로 성장시키고 싶은지
- 지도 가능 분야
- 출강 가능 형태
- 간단한 이력/자격/강의 철학 카드 구성

[교육 철학 페이지에 들어갈 내용]
반드시 단순 소개문이 아니라, 한 편의 교육 선언문처럼 설계해라.
포함 요소:
- 왜 질문형 수업인가
- 왜 토론과 코딩을 함께 하는가
- 왜 AI 교육에 철학과 윤리가 필요한가
- 아이들이 ‘정답을 맞히는 사람’이 아니라 ‘생각하는 사람’이 되어야 하는 이유
- 아동심리학, 교육심리학, 아들러 개인심리학을 교육에 녹이는 관점
- 질문 → 토론 → 코딩 → 해석 → 성찰의 수업 흐름
- 학부모/학교 담당자가 봐도 신뢰할 수 있는 문장력으로 작성

[프로그램 페이지 구성]
프로그램 목록 페이지와 상세 페이지를 각각 구성해라.
최소 아래 프로그램 카테고리를 반영해라.

- AI 블록코딩 입문
- 생각하는 Codiny 수업
- 파이썬 기초
- 생성형 AI 첫걸음
- 머신러닝 입문
- 딥러닝/AI 이해 수업
- AICE Future 대비반
- AICE Basic 로드맵
- 학교/기관 맞춤형 특강

각 프로그램 상세 페이지에는 아래를 넣어라.
- 대상 학년
- 수업 목표
- 핵심 개념
- 대표 활동
- 준비물
- 차시 예시
- 기대 효과
- 추천 대상
- 문의 연결 버튼

[수업실/클래스룸 페이지]
이 페이지는 PPT를 대체할 수 있는 웹 수업 공간처럼 보여야 한다.
단순 게시글 목록처럼 만들지 말고, “실제 수업 흐름”이 보이도록 만들어라.

권장 구조:
- 오늘의 질문
- 상황 제시
- 생각하기
- 토론 포인트
- 실습 미션
- 예시 결과
- 정리 질문
- 성찰 기록

최소 3개의 샘플 수업을 만들어라.
예시 주제:
1. 트롤리 딜레마와 AI의 선택
2. AI는 친구가 될 수 있을까?
3. 데이터를 믿어야 할까?

각 샘플 수업에는 아래를 포함해라.
- 도입 질문
- 수업 목표
- 활동 흐름
- Codiny 또는 Python 연결 포인트
- 정리 문항
- 교사용 의도 또는 수업 포인트(노출/비노출 구조는 자유)
- 시각적으로 발표 자료처럼 보기 좋은 구성

[질문 연구소 페이지]
이 페이지는 이 사이트의 차별화 포인트다.
질문형 학습을 대표하는 아카이브 공간으로 설계해라.

카테고리 예시:
- 철학 질문
- 과학 질문
- 논리 질문
- 윤리 질문
- AI 시대의 인간 질문

최소 15~20개의 예시 질문 카드를 만들어라.
예시 질문:
- AI가 사람보다 더 정확하면 사람의 판단은 필요 없을까?
- 모두를 만족시키는 선택은 가능할까?
- 감정을 이해하는 기계는 살아 있다고 할 수 있을까?
- 데이터가 많으면 항상 더 좋은 결정을 할 수 있을까?
- 정답이 여러 개인 문제는 어떻게 풀어야 할까?

[자료실 페이지]
실제 운영 가능한 자료실처럼 구성해라.
포함 예시:
- 활동지
- 질문 카드 PDF
- 수업 요약 자료
- AICE 준비자료
- 학부모 안내자료
- 기관 제안서
- 데모 다운로드 항목 또는 샘플 링크
초기에는 mock 데이터로 구성해도 된다.

[프로젝트/성과/갤러리 페이지]
- 학생 결과물 예시
- 수업 활동 사진 자리
- 블록코딩 결과 예시
- 파이썬 실습 결과 예시
- 프로젝트 카드
- 향후 실제 사례가 추가될 수 있는 구조
- 비어 보이지 않도록 샘플 데이터와 레이아웃을 채워라

[학교·기관 제안 페이지]
이 페이지는 담당자가 바로 검토할 수 있어야 한다.
포함 내용:
- 어떤 학교/기관에 적합한지
- 운영 가능한 방식(특강/정규/캠프/방과후 등)
- 대상 학년
- 예상 운영 흐름
- 기대 효과
- 문의 절차
- 강사 강점
- 프로그램 제안서 요청 CTA

[후기/기대효과 페이지]
실제 후기가 아직 없더라도 비어 두지 말고, 초기에는 기대효과 중심으로 구성해라.
포함 내용:
- 사고력 향상
- 문제해결력
- 표현력
- 토론 태도
- AI 이해도
- 자격증 준비 역량
- 수업의 차별점
후기 mock 카드 또는 향후 후기 입력 가능한 구조도 고려해라.

[문의 페이지]
반드시 실제 폼처럼 설계해라.
필수 입력 항목 예시:
- 이름
- 학교/기관명
- 연락처
- 이메일
- 대상 학년
- 예상 인원
- 희망 수업 주제
- 희망 일정
- 문의 내용
- 개인정보 동의 체크
폼 유효성 검사와 제출 완료 UI도 고려해라.
실제 백엔드가 없으면 로컬 상태/목업 저장 구조라도 만들어라.

[관리자 구조]
관리자 페이지 전체 구현이 어렵더라도 최소한 구조와 기본 화면은 설계해라.
포함하면 좋은 메뉴:

기존 코드가 있으면 그것을 바탕으로 보강하고, 없으면 처음부터 필요한 파일을 생성해라.
```

## Antigravity에 함께 주면 좋은 보조 지시문

필요하면 아래 보조 지시문도 추가로 붙여서 사용할 수 있다.

### 1) 확인 질문 최소화 강화용
```text
중간 확인은 꼭 필요한 경우에만 1회 이하로 제한하고, 나머지는 스스로 합리적으로 판단해서 진행해라. 사용자가 이미 원하는 방향을 충분히 설명했으므로, 더 이상 사소한 결정을 물어보지 말고 구현 중심으로 움직여라.
```

### 2) 콘텐츠 빈칸 방지용
```text
페이지를 틀만 만들지 말고 실제 운영 중인 사이트처럼 보이도록 샘플 문구, 카드, 자료, 프로그램 설명, 질문 카드, 수업 예시까지 충분히 채워라.
```

### 3) 구조 완성도 강화용
```text
공통 컴포넌트, 데이터 구조, 페이지 간 연결, CTA 흐름, 문의 동선, 관리자 확장성까지 고려해서 실제 제품처럼 설계해라.
```

### 4) 기존 프로젝트 보강용
```text
기존 파일을 먼저 읽고, 이미 구현된 부분은 재사용하되 전체 톤과 구조가 어색하면 과감하게 리팩터링해라. 단, 프로젝트가 깨지지 않도록 단계적으로 수정해라.
```

```text
현재 프로젝트는 이미 일부 페이지가 만들어져 있으니, 기존 구조를 분석한 후 부족한 콘텐츠와 페이지를 채우고 전체 완성도를 올리는 방향으로 작업해라.
```

### 방식 C: 홈페이지는 이미 있고 내용만 채우기
- 실제 프롬프트 전체에서 기술 스택 관련 부분은 유지

예시:

```text
새 사이트를 처음부터 만들기보다, 현재 존재하는 홈페이지 구조를 유지하면서 콘텐츠 기획, 카피라이팅, 페이지 구성 보완, 프로그램 페이지와 수업실 콘텐츠 추가에 집중해라.
```
